4 Plano de Teste e Estratégia
4.1 Teste funcional

O objetivo do teste funcional é garantir que todo o produto de software funcione
de acordo com os requisitos, e nenhum erro significativo aparece no aplicativo.
O teste funcional é a parte mais substancial do teste de software. envolve verificar
diferentes aspectos do sistema. Um produto de software deve passar em todos os testes planejados.
Apenas neste caso, sua qualidade pode ser assegurada.

- Técnica:
Executar cada caso de uso, fluxo de caso de uso ou função, usando
dados válidos e inválidos, para verificar o seguinte:
● Os resultados esperados ocorrem quando dados válidos são usados.
● As mensagens de erro ou aviso apropriadas são exibidas quando
dados inválidos são usados.
● Cada regra é aplicada corretamente.

- Critério de Entrada:
● A construção do aplicativo está concluída.
● Os dispositivos, instrumentos e outros equipamentos necessários.
● O ambiente de teste é preparado e o aplicativo é liberado para o
ambiente de teste.

- Critérios de Conclusão:
● Todos os testes planejados são executados.
● Não há erros que interrompam a exibição.
● Todos os erros de alta prioridade e gravidade são corrigidos.
● Os resultados dos testes são avaliados, discutidos e aprovados.

4.2 Relatório de erros
Os relatórios de bugs são criados para fornecer à equipe de desenvolvimento e ao projeto
gerentes com informações exaustivas sobre os defeitos descobertos. Eles devem ser úteis
determinar as causas dos erros e corrigi-los.
